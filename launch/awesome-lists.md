# Awesome lists

On-topic curated lists where fable-skills genuinely belongs (it's a set of Claude Code
skills with tests). **Verify each repo's CONTRIBUTING before submitting** — several of these
have non-obvious, automated submission flows, and getting it wrong wastes a maintainer's
time. Submit one resource per list, with a one-line description that matches the README.

Canonical one-liner to reuse:

> **fable-skills** — Six Claude Code skills that harden Opus 4.8 toward frontier behavior
> (claims, scope, reporting, code idiom). Written by Fable 5, pressure-tested on the target
> model with transcripts included. MIT.

Repo: `https://github.com/DizzyMii/fable-skills`

---

### 1. hesreallyhim/awesome-claude-code  ⚠️ does NOT take submission PRs

The flagship list. Submissions go through a **GitHub Issue**, not a pull request — the repo
states only the maintainer's automation opens PRs. Open a new issue using their **`[Resource]`
submission template** and fill its fields. Title format: `[Resource]: fable-skills`. Do not
open a PR.
→ https://github.com/hesreallyhim/awesome-claude-code/issues/new/choose

### 2. travisvn/awesome-claude-skills

Skills-specific list. Standard fork → add entry → PR. One skill/resource per PR, clear title,
check for duplicates, follow the existing line format in the README. Read its CONTRIBUTING.md
first.
→ https://github.com/travisvn/awesome-claude-skills

### 3. ComposioHQ/awesome-claude-skills

Curated Claude Skills list. Fork, add your entry in the matching section, PR with a clear
description. Confirm the section/category before submitting.
→ https://github.com/ComposioHQ/awesome-claude-skills

### 4. rohitg00/awesome-claude-code-toolkit

Has a structured CONTRIBUTING: feature branch (`feature/...`), commit like
`Add fable-skills: <description>`, push, PR. Note this list expects some resources placed in
directories — confirm whether a link entry or a directory contribution is wanted, and say so
in the PR.
→ https://github.com/rohitg00/awesome-claude-code-toolkit

### 5. dykyi-roman/awesome-claude-code

Has a CONTRIBUTING.md; standard PR flow against the `master` branch. Match the existing entry
format.
→ https://github.com/dykyi-roman/awesome-claude-code

### 6. sindresorhus/awesome (and the general "awesome" ecosystem)

Only relevant if fable-skills is bundled into a broader "awesome-claude" entry that itself
meets sindresorhus/awesome's strict bar (minimum age, README quality, CI). Not a day-one
target — revisit once the repo has traction and a few contributors.

---

**Etiquette:** PR descriptions should say what it is in one line and link the README. Don't
re-submit to a list that rejected it. If a list is dormant (no merges in months), skip it —
a PR into a dead list buys nothing. Space submissions out; a burst of identical PRs across
lists on the same hour reads as spam.
